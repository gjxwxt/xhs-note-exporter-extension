# XHS Note Capture

一个更稳的 Chrome 扩展版本，适合在小红书网页版里采集帖子详情页，并把多条记录汇总到同一个任务里，最后统一导出 Excel。

用途：

- 手动采集当前打开的帖子详情页
- 开启自动采集后，监听帖子详情页路由变化，自动采集未入库的新帖子
- 同一个采集任务里持续追加多条记录
- 采集封面图、账号/昵称、发布时间、标题、正文、点赞数、评论数、笔记类型、笔记链接、抓取状态、备注
- 自动采集遇到已采集过的帖子时会自动跳过
- 手动重复采集同一篇帖子时，按帖子链接优先去重并更新原记录
- 支持高清导出和快速导出两种 `.xlsx`

## 安装

1. 打开 Chrome。
2. 进入 `chrome://extensions/`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择当前这个文件夹：

```text
/Users/imac/Documents/code/claude-code-workspace/xhs-note-exporter-extension
```

6. 如果你已经提前打开了小红书页面，安装完扩展后记得刷新一下页面。

## 使用

1. 登录你自己的小红书网页版。
2. 点击扩展图标。
3. 如果还没有采集任务，先点击“新建任务”。
4. 可选其一：
5. 点击“采集当前帖子”，手动采集当前详情页。
6. 点击“开启自动采集”，之后正常打开新的帖子详情页即可自动入库。
7. 全部采集完成后，按需选择“高清导出”或“快速导出”。
8. 如需清空当前任务、删除单条记录或删除整个任务，可直接在 popup 中操作。

## 导出结构

明细 sheet 固定字段：

- `封面图`
- `序号`
- `账号/昵称`
- `发布时间`
- `标题`
- `正文`
- `点赞数`
- `评论数`
- `笔记类型`
- `笔记链接`
- `抓取状态`
- `备注`

此外还会附带：

- `月度汇总` sheet
- `采集信息` sheet

## 当前实现说明

- 当前版本不再自动滚动账号列表，也不再自动帮你点开帖子。
- 自动采集只负责监听已经打开的帖子详情页，不会代替你浏览页面。
- 采集任务相当于“待导出的数据容器”，不是直接实时改写本地 Excel。
- 真正生成 `.xlsx` 发生在你点击导出按钮时。
- Excel 里的封面图使用帖子图片原图链接下载，并以较高分辨率嵌入。
- 快速导出不包含封面图，适合 100 条以上的大批量数据。

## 已知限制

- 自动采集要在你已经选中采集任务并开启开关后才会生效。
- 小红书页面结构变动后，可能需要微调选择器。
- 如果帖子详情页没完全加载出来，可能会记成“部分成功”并写进备注。
- 如果内容是私密、半私密或触发平台风控，插件无法绕过平台限制。
